# Computational Modeling of Belief Updating and Credibility

## Overview
This repository contains the MATLAB codebase and datasets for the paper: **"Disinformation elicits learning biases"**. 
You can read the preprint [here](https://elifesciences.org/reviewed-preprints/106073#s2)

The pipeline covers the full modeling workflow used in the study: generative simulations, parameter estimation via maximum likelihood, mixed-effects statistical modeling of behavior, and final manuscript visualizations.

## Data Structure & Variables

The project data is divided across primary `.mat` files:

### 1. `data.mat`
Contains both the empirical behavioral data and the generative simulated datasets used for parameter recovery and model validation.
* **`ordered_data`**: A `204x3` cell array containing empirical data for 204 subjects. 
    * **Column 1**: Subject ID
    * **Column 2**: The core behavioral `struct` for each subject, containing the following trial-by-trial arrays:
        * `pick`: The option chosen by the subject (2 if best, 1 if worst).
        * `probs`: The corresponding reward probabilities of each bandit.
        * `agent`: The identity of the agent providing feedback (1, 2, or 3).
        * `real`: The objective credibility/reliability of the agent (0.5, 0.75, or 1).
        * `reward`: The true outcome of the trial (0 for no reward, 1 for reward).
        * `feedback`: The feedback provided by the agent (0 for false/negative, 1 for true/positive).
        * `timeouts`: Indicates if the subject failed to respond in time (1 if there has been a timeout trial, 0 otherwise).
        * `block`: The current block number.
        * `game`: The current bandit pair number.
        * `response_time`: The time taken to make a choice.
    * **Column 3**: Interleaved data (with 3 bandit pairs interleaved per block). When analyzing data from this column, it is important to refer to the `game` variable, which codes exactly which bandit pair is presented in a given trial.

* **Simulated Datasets (`1020x3` cell arrays)**: 
    * `sim_data_bay_ideal`: Data simulated using the Ideal Bayesian model.
    * `sim_data_bay_cred`: Data simulated using the Credibility Bayesian model.
    * `sim_data_cred`, `sim_data_credval`, `sim_data_truth`, `sim_data_null`: Data simulated from their respective computational models.
    * *Structure Note*: These arrays contain 5 simulation runs per subject (204 subjects × 5 = 1020 rows).

### 2. `parameters.mat`
Contains the parameters obtained from direct fits to the empirical `ordered_data`.
* **Parameter Matrices** (`parameters_[model name]`): 2D matrices (structured as `N_subj x #par`) for each fitted model. Each row is a single subject, and columns correspond to the specific fitted parameters.
* **Evaluation Metrics** (`fval_[model name]`) : Arrays of negative loglikelihoods, structured as `N_subj x 1` vectors.

### 3. `parameters_baysim.mat` *(Generated by cross-fitting scripts)*
Contains the parameter recovery results obtained by fitting the standard computational models to the Bayesian simulated datasets.
* **Parameter Matrices (`N_subj x #par x 5`)** (`params_sim[bayesian sim]_fit[model fit]`) : Restructured 3D matrices containing the fitted parameters across the 5 simulation repetitions.
* **Evaluation Matrices (`N_subj x 5`)** (`fval_sim[bayesian sim]_fit[model fit]`) : Restructured 2D matrices containing the negative loglikelihoods of model fits.

## Core Functions

### 1. Simulation Functions
* **`simulate_models.m`**: Generates synthetic trial-by-trial behavioral data based on the standard fitted models (e.g., `cred`, `cred-val`, `truth`). Used to create the 1020-row arrays for parameter recovery.
* **`simulate_bayesian_models.m`**: Generates synthetic data using the Bayesian architectures (`Ideal` and `Credibility`).

### 2. Model Fitting Functions
* **`fit_models.m`**: The core optimization function for standard models. Takes a data cell array and a string identifier (e.g., `'cred'`, `'cred-val'`, `'truth'`) to find the best-fitting parameters using maximum likelihood estimation.
* **`fit_bayesian_models.m`**: Specifically handles the parameter estimation and optimization for the Bayesian model variants, extracting metrics like subjective priors and credibility integration weights.

### 3. Statistical Analysis (Mixed-Effects Models)
* **`MEM_choiceRepetition.m`**: Runs Mixed-Effects Models to analyze trial-by-trial choice repetition behavior, assessing how previous feedback and agent credibility influence subsequent decisions.
* **`MEM_contextEffect.m`**: Evaluates contextual behavioral effects using mixed-effects modeling, likely isolating how the surrounding task environment or concurrent agents impact choice dynamics.
* **`MEM_parameters.m`**: Conducts group-level mixed-effects statistical analyses directly on the fitted computational parameters extracted from `parameters.mat`.

### 4. Visualization & Plotting
* **`plot_Parameters.m`**: Generates the main manuscript figures comparing model parameters (e.g., scatter plots for parameter recovery, bar plots for empirical parameter differences across conditions).

## Reproducing Figures & Stats

**Figure 3: Choice Repetition & Parameters**
* **Fig 3a (Plot & Stats):** `MEM_choiceRepetition(ordered_data)`
* **Fig 3b (Plot & Stats):** `MEM_choiceRepetition(sims)` *(Note: swap `sims` for your specific simulated dataset)*
* **Fig 3c (Plot):** `plot_Parameters(parameters_cred,'cred')` 
* **Fig 3c (Stats):** `mdl = MEM_parameters(parameters_cred, 1)`
* **Fig 3d (Plot):** `plot_Parameters(parameters_bay_cred,'bay_cred')` 

**Figure 4: Context Effects**
* **Fig 4b (Plot & Stats):** `MEM_contextEffect(ordered_data)`

**Figure 5: Valence Bias**
* **Fig 5 (Plot):** `plot_Parameters(parameters_credval,'cred-val')` *(Note: you can switch the parameters from the ones of bayesian simulations)*
* **Fig 5 (Stats - Cred-Val CA):** `mdl = MEM_parameters(parameters_credval, 2)`
* **Fig 5 (Stats - rVBI):** `mdl = MEM_parameters(parameters_credval, 3)`

**Figure 6: Truth Inference Analysis**
* **Fig 6c (Plot):** `plot_Parameters(parameters_truth,'truth')` *(Note: you can switch the parameters from the ones of bayesian simulations)*
