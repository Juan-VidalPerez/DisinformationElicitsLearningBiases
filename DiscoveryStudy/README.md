# Computational Modeling of Belief Updating and Credibility (Pilot Study)

## Overview
This repository contains the MATLAB codebase and datasets for the **Discovery Study** of the paper: **"Disinformation elicits learning biases"**. 
You can read the preprint [here](https://elifesciences.org/reviewed-preprints/106073#s2) (see SI 1).

The pipeline covers the full modeling workflow used in the discovery study (which features a **blocked design** and **4 agents**): generative simulations, parameter estimation via maximum likelihood, mixed-effects statistical modeling of behavior, and visualizations.

## Data Structure & Variables

The project data is divided across primary `.mat` files:

### 1. `data.mat`
Contains both the empirical behavioral data and the generative simulated datasets used for parameter recovery and model validation.
* **`ordered_data`**: An `N_subj x 2` cell array containing empirical data for the discovery study subjects. 
    * **Column 1**: Subject ID
    * **Column 2**: The core behavioral `struct` for each subject, containing strictly blocked trial-by-trial arrays:
        * `pick`: The option chosen by the subject (2 if best, 1 if worst).
        * `Prew`: The corresponding prior reward probabilities of each bandit.
        * `agent`: The identity of the agent providing feedback (1, 2, 3, or 4).
        * `reliability` / `rel`: The objective credibility of the agent (0.5, 0.7, 0.85, or 1.0).
        * `condition`: Block type (1 = Informative with 75/25% probs, 0 = Control with equal probs uniformly sampled between 60 and 80%).
        * `reward`: The true outcome of the trial (0 for no reward, 1 for reward).
        * `feedback`: The feedback provided by the agent (0 for false/negative, 1 for true/positive).
        * `accuracy`: 1 if the objectively better option was chosen, 0 if not, and 0.5 during control blocks.
        * `timeouts`: Indicates if the subject failed to respond in time (1 for timeout, 0 otherwise).
        * `block`: The current block number (1 to 15).
        * `trial`: The trial number within the current block.
        * `response_time`: The time taken to make a choice.

* **Simulated Datasets (`(N_subj*5) x 2` cell arrays)**: 
    * `sim_data_bay_ideal`: Data simulated using the Ideal Bayesian model.
    * `sim_data_bay_cred`: Data simulated using the Credibility Bayesian model.
    * `sim_data_cred`, `sim_data_credval`, `sim_data_truth`, `sim_data_null`: Data simulated from their respective computational models.
    * *Structure Note*: These arrays contain 5 simulation runs per subject.

### 2. `parameters.mat`
Contains the parameters obtained from direct fits to the empirical `ordered_data`.
* **Parameter Matrices** (`parameters_[model name]`): 2D matrices (structured as `N_subj x #par`) for each fitted model. Each row is a single subject, and columns correspond to the specific fitted parameters.
* **Evaluation Metrics** (`fval_[model name]`) : Arrays of negative loglikelihoods, structured as `N_subj x 1` vectors.

### 3. `parameters_baysim.mat` *(Generated by cross-fitting scripts)*
Contains the parameter recovery results obtained by fitting the standard computational models to the Bayesian simulated datasets.
* **Parameter Matrices (`N_subj x #par x 5`)** (`params_sim[bayesian sim]_fit[model fit]`) : Restructured 3D matrices containing the fitted parameters across the 5 simulation repetitions.
* **Evaluation Matrices (`N_subj x 5`)** (`fval_sim[bayesian sim]_fit[model fit]`) : Restructured 2D matrices containing the negative loglikelihoods of model fits.

## Core Functions

### 1. Simulation Functions
* **`simulate_models.m`**: Generates synthetic trial-by-trial behavioral data based on the standard fitted models (e.g., `cred`, `cred-val`, `truth`) adapted for 4 agents and blocked bandit pairs conditions. 
* **`simulate_bayesian_models.m`**: Generates synthetic data using the Bayesian architectures (`ideal` and `cred`).

### 2. Model Fitting Functions
* **`fit_models.m`**: The core optimization function for standard models. Takes a 2-column data cell array and a string identifier (e.g., `'cred'`, `'cred-val'`, `'truth'`) to find the best-fitting parameters using maximum likelihood estimation.
* **`fit_bayesian_models.m`**: Handles the parameter estimation and optimization for the Bayesian model variants, scaling bounds and priors for the 4-agent structure.

### 3. Statistical Analysis (Mixed-Effects Models)
* **`MEM_choiceRepetition.m`**: Runs Mixed-Effects Models to analyze trial-by-trial choice repetition behavior across the 4 agents, properly accounting for both informative and control blocks.
* **`MEM_parameters.m`**: Conducts group-level mixed-effects statistical analyses directly on the fitted computational parameters, updated with dummy variables for the 4-agent discovery study design.

### 4. Visualization & Plotting
* **`plot_Parameters.m`**: Generates the figures comparing model parameters across the 4 agents using an updated 4-color palette.

## Reproducing Figures & Stats

**SI Figure 2: Choice Repetition & Parameters**
* **Fig S2a (Plot & Stats):** `MEM_choiceRepetition(ordered_data)`
* **Fig S2b (Plot & Stats):** `MEM_choiceRepetition(sims)` *(Note: swap `sims` for your specific simulated dataset)*
* **Fig S2c (Plot):** `plot_Parameters(parameters_cred,'cred')` 
* **Fig S2c (Stats):** `mdl = MEM_parameters(parameters_cred, 1)`
* **Fig S2d (Plot):** `plot_Parameters(parameters_bay_cred,'bay_cred')` 

**SI Figure 3: Valence Bias**
* **Fig S3 (Plot):** `plot_Parameters(parameters_credval,'cred-val')` *(Note: you can switch the parameters from the ones of bayesian simulations)*
* **Fig S3 (Stats - Cred-Val CA):** `mdl = MEM_parameters(parameters_credval, 2)`
* **Fig S3 (Stats - rVBI):** `mdl = MEM_parameters(parameters_credval, 3)`

**Figure S4: Truth Inference Analysis**
* **Fig S4c (Plot):** `plot_Parameters(parameters_truth,'truth')` *(Note: you can switch the parameters from the ones of bayesian simulations)*
